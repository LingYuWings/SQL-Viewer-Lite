# SQL-Viewer Lite — 开发计划

## 总览

本项目分三个阶段迭代开发，每个阶段产出可运行的版本。各阶段之间存在依赖关系，后续阶段基于前一阶段的代码基础演进。

| 阶段 | 目标 | 预估周期 | 前置条件 |
|------|------|---------|---------|
| **第一阶段** | 基础功能可用 | 2~3 周 | 无 |
| **第二阶段** | 体验优化 | 2~3 周 | 第一阶段完成 |
| **第三阶段** | 打包发布 | 1~2 周 | 第二阶段完成 |

---

## 第一阶段：基础功能可用

**目标**：跑通核心功能链路——登录 → 浏览 → 查看 → 编辑，UI 从简，确保功能正确。

### 里程碑

| # | 里程碑 | 交付物 |
|---|--------|--------|
| M1 | 项目骨架搭建 | 目录结构、依赖清单、入口文件、基础 QSS |
| M2 | 登录模块完成 | 登录窗口、连接管理、多连接保存 |
| M3 | 数据库浏览完成 | 左侧数据库/表树、搜索过滤 |
| M4 | 数据查看完成 | 表格分页加载、排序、列筛选 |
| M5 | 数据编辑完成 | 单元格编辑、新增/删除行、事务提交 |
| M6 | 多线程集成 | Worker 线程池、异步加载、取消操作 |
| M7 | CLI 后端 + 单元测试 | 命令行接口、核心模块测试覆盖 |

### 详细任务拆解

#### M1 — 项目骨架

- 创建 `sql_viewer_lite/` 包结构（ui/ core/ models/ utils/ backend/ tests/）
- 编写 `requirements.txt`（PyQt5, PyMySQL, pycryptodome 等）
- 实现 `main.py` 入口，启动空白主窗口
- 编写基础 QSS 样式文件 `styles/main.qss`

#### M2 — 登录模块

- 开发 `login_window.py`：主机/端口/用户名/密码输入表单
- 实现 `db_connection.py`：PyMySQL 连接封装，支持连接测试
- 实现 `config_manager.py`：连接配置 JSON 持久化存储
- 实现 `encryption.py`：密码 AES 加密/解密
- 登录成功 → 发射信号，传递连接对象

#### M3 — 数据库浏览

- 实现 `main_window.py` 左侧 QTreeWidget：展示数据库 → 表层级
- 实现数据库/表搜索过滤（QLineEdit + QSortFilterProxyModel）
- 双击表名 → 发射信号，请求加载数据
- 展示表元信息（行数、引擎、字符集）——通过 `SHOW TABLE STATUS`

#### M4 — 数据查看

- 实现 `table_view.py`：QTableWidget 分页数据展示
- 分页逻辑：每页 500 行，支持翻页
- 列排序：点击列头触发 `ORDER BY` 查询
- 列筛选：每列顶部输入框，实时过滤
- 实现 `table_structure.py`：展示字段名、类型、约束、索引

#### M5 — 数据编辑

- QTableWidget 启用 `setEditTriggers` 支持单元格直接编辑
- 工具栏新增"新增行"、"删除行"按钮
- 编辑数据暂存到内存 diff 模型
- "提交"按钮 → 构建 `UPDATE`/`INSERT`/`DELETE` SQL → 事务执行
- "撤销"按钮 → 丢弃未提交的 diff

#### M6 — 多线程集成

- 实现 `db_worker.py`：基于 QThread 的查询工作者
- 定义信号：`result_ready`, `error_occurred`, `progress_updated`
- 主窗口通过信号调度 Worker，避免主线程阻塞
- 实现查询取消机制（Worker 检查 `is_cancelled` 标志）
- 多表同时加载：每个表一个独立 Worker

#### M7 — CLI 后端 + 测试

- 实现 `backend/cli.py`：命令行参数解析，支持 `--host`, `--user`, `--query` 等
- CLI 支持：列出数据库、列出表、执行 SQL、导出 CSV
- 编写 `test_db_connection.py`：连接/断开/错误处理测试
- 编写 `test_query_executor.py`：查询执行/事务测试
- 编写 `test_config_manager.py`：配置读写测试

### 第一阶段验收标准

- [ ] 能成功登录 MySQL 并加载数据库列表
- [ ] 能浏览数据库/表，双击打开查看数据
- [ ] 大表分页加载不卡顿
- [ ] 能编辑单元格、新增/删除行并提交
- [ ] 多表同时加载时主界面保持响应
- [ ] CLI 后端能独立执行查询并导出
- [ ] 核心模块单元测试通过

## 第二阶段：体验优化

**目标**：在功能完整的基础上，提升 UI 美观度、交互流畅度和功能完善度。

### 里程碑

| # | 里程碑 | 交付物 |
|---|--------|--------|
| M8 | UI 全面美化 | 深色/浅色主题、现代化布局、响应式设计 |
| M9 | 功能增强 | SQL 执行器、数据导出、右键菜单 |
| M10 | 国际化 | 中英文切换 |
| M11 | SSH 隧道 | 支持通过 SSH 隧道连接远程 MySQL |
| M12 | 性能优化 | 连接池、查询缓存、内存优化 |

### 详细任务拆解

#### M8 — UI 全面美化

- 重写 QSS：深色主题 + 浅色主题双套样式
- 实现主题切换：设置菜单中一键切换
- 主窗口布局优化：可拖拽分割线、可折叠侧边栏
- 登录窗口美化：卡片式居中布局，Logo 和品牌展示
- 加载状态：骨架屏 / Spinner 动画
- 状态栏信息完善：连接状态、当前表、行数、查询耗时

#### M9 — 功能增强

- 实现 SQL 输入框（DockWidget）：手动输入 SQL、语法高亮（可选）
- 查询历史记录面板
- 数据导出：选中行导出为 CSV / JSON / SQL INSERT
- 右键菜单：复制单元格、复制行、刷新、导出
- 表搜索增强：支持模糊匹配、正则匹配

#### M10 — 国际化

- 使用 Qt 的 `QTranslator` 机制
- 提取可翻译字符串到 `.ts` 文件
- 编写中文（`zh_CN.ts`）和英文（`en_US.ts`）翻译
- 设置菜单中提供语言切换选项

#### M11 — SSH 隧道

- 集成 `paramiko` 库实现 SSH 隧道
- 登录窗口增加 SSH 隧道选项（折叠面板）
- 支持密码和密钥文件两种认证方式
- 通过隧道转发端口连接远程 MySQL

#### M12 — 性能优化

- 实现数据库连接池（复用连接，避免频繁创建/销毁）
- 查询结果缓存（相同查询 5 秒内复用）
- 大数据量场景内存优化（使用懒加载 + 虚拟滚动）
- 性能基准测试脚本

### 第二阶段验收标准

- [ ] 深色/浅色主题切换正常
- [ ] 所有界面元素响应式布局，缩放无错位
- [ ] SQL 执行器能执行查询并展示结果
- [ ] 数据导出 CSV / JSON / SQL 正确
- [ ] 中英文切换完整
- [ ] SSH 隧道连接远程 MySQL 成功
- [ ] 大表（>10万行）首屏加载 < 3 秒

## 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| PyQt5 大数据表格渲染性能不足 | 高 | 使用虚拟滚动或考虑切换到 `QTableView` + 自定义 Model |
| PyMySQL 不支持某些 MySQL 特性 | 中 | 必要时降级到 `mysqlclient` |
| PyInstaller 打包体积过大 | 中 | 使用 `--exclude-module` 排除无用依赖，压缩资源 |
| SSH 隧道稳定性 | 中 | 超时重连机制，连接状态监控 |
| 跨平台兼容性（Win/Mac） | 中 | 早期就在双平台测试，避免后期大量修复 |
