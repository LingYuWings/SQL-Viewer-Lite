# SQL-Viewer Lite — 项目规格说明书

## 1. 项目概述

**SQL-Viewer Lite** 是一款轻量、高速的 MySQL 数据查看与编辑工具，基于 Python 桌面技术栈开发。旨在为开发者和数据管理员提供一个简洁、高效的数据库浏览体验，支持多线程并发查询、数据编辑和现代化 UI 设计。

### 1.1 项目目标

- 提供类 Navicat / Excel 的多线程数据库浏览与编辑体验
- 使用 QSS 实现现代化、前端风格的用户界面
- 保持轻量级，启动快、占用资源少
- 同时提供 GUI 和 CLI/后端接口，方便测试和集成

### 1.2 技术栈

| 层级 | 技术选型 |
|------|---------|
| 编程语言 | Python 3.8+ |
| GUI 框架 | PyQt5 |
| 数据库驱动 | PyMySQL |
| 样式方案 | QSS (Qt Style Sheets) |
| 并发模型 | QThread / Python threading |
| 构建/打包 | PyInstaller (第三阶段) |

## 2. 核心功能

### 2.1 用户登录

- 用户通过输入 MySQL 连接信息进行身份验证（主机、端口、用户名、密码）
- 登录成功后获取服务器上的数据库列表
- 支持保存/管理多个连接配置（本地加密存储）
- 连接失败时提供清晰的错误提示

### 2.2 数据库浏览

- 登录后展示当前 MySQL 服务器上所有可用数据库
- 用户选择一个数据库后，加载该库下所有数据表
- 支持数据库/表的搜索和筛选
- 展示表的基本信息（表名、行数、引擎、字符集等）

### 2.3 数据查看

- 以表格形式展示选中表的数据内容
- 支持分页加载，避免大数据表一次性加载导致卡顿
- 支持列排序（升序/降序）
- 支持按列筛选/搜索
- 支持查看表结构（字段名、类型、约束、索引等）

### 2.4 数据编辑

- 支持直接在表格中编辑单元格数据
- 支持新增行、删除行操作
- 支持批量编辑后统一提交
- 编辑前自动备份原始数据，支持撤销
- 所有写操作通过事务处理，确保数据一致性

### 2.5 多线程查询

- 多个表的数据加载和搜索采用多线程并行执行
- 使用 QThread + Signal/Slot 机制保证线程安全
- 主界面保持响应，不因后台查询而阻塞
- 支持查询任务的取消操作

### 2.6 SQL 执行器（可选/扩展）

- 提供简易 SQL 输入框，支持手动执行自定义 SQL
- 展示查询结果和执行信息
- 支持查询历史记录

## 3. 架构设计

### 3.1 模块划分

```
sql_viewer_lite/
├── main.py                 # 程序入口
├── ui/                     # UI 相关模块
│   ├── __init__.py
│   ├── login_window.py     # 登录窗口
│   ├── main_window.py      # 主窗口（数据库/表浏览 + 数据展示）
│   ├── table_view.py       # 数据表格视图组件
│   ├── table_structure.py  # 表结构查看组件
│   └── styles/             # QSS 样式文件
│       └── main.qss
├── core/                   # 核心业务逻辑
│   ├── __init__.py
│   ├── db_connection.py    # 数据库连接管理
│   ├── db_worker.py        # 多线程查询工作者
│   └── query_executor.py   # SQL 执行器
├── models/                 # 数据模型
│   ├── __init__.py
│   ├── connection.py       # 连接配置模型
│   └── table_meta.py       # 表元数据模型
├── utils/                  # 工具函数
│   ├── __init__.py
│   ├── config_manager.py   # 连接配置的持久化管理
│   └── encryption.py       # 密码加密存储
├── backend/                # 非 GUI 后端（用于测试）
│   ├── __init__.py
│   └── cli.py              # 命令行接口
├── tests/                  # 单元测试
│   ├── __init__.py
│   ├── test_db_connection.py
│   └── test_query_executor.py
├── requirements.txt
└── SPEC.md
```

### 3.2 数据流

```
用户操作 → UI 层 → 信号发射 → Worker 线程 → 数据库查询 → 结果返回 → UI 更新
```

### 3.3 关键设计原则

- **MVC 分层**：UI 层、业务逻辑层、数据访问层分离
- **信号驱动**：跨线程通信统一使用 Qt Signal/Slot
- **单例连接**：数据库连接池统一管理，避免连接泄漏
- **延迟加载**：大表数据分页加载，元数据按需获取

## 4. UI 设计规范

### 4.1 整体风格

- 采用 QSS 实现现代化扁平设计，参考前端 UI 框架风格
- 主色调：深色主题（可选浅色主题）
- 圆角卡片式布局，适当的阴影和间距
- 响应式布局，支持窗口缩放

### 4.2 界面布局

```
┌─────────────────────────────────────────────────────┐
│  工具栏：连接状态 | 数据库选择 | 搜索 | 刷新 | 设置  │
├────────────────┬────────────────────────────────────┤
│                │                                    │
│   侧边栏       │         数据表格区域                │
│   - 数据库列表  │   (多 Tab 页，每个表一个 Tab)       │
│   - 表列表     │                                    │
│   - 表结构     │                                    │
│                │                                    │
├────────────────┴────────────────────────────────────┤
│  状态栏：连接信息 | 当前表 | 行数 | 查询耗时          │
└─────────────────────────────────────────────────────┘
```

### 4.3 交互规范

- 登录窗口居中显示，支持记住连接信息
- 左侧数据库/表树形结构支持搜索过滤
- 数据表格支持右键菜单（复制、导出、刷新等）
- 加载数据时显示进度指示器
- 操作结果通过 Toast / 状态栏提示反馈

---

## 5. 安全性

- 密码在本地存储时使用加密（如 AES 或 keyring 系统密钥库）
- 连接信息不明文写入配置文件
- SQL 执行器限制危险操作（如 DROP、TRUNCATE），需二次确认
- 连接超时自动断开，避免长连接资源占用

---

## 6. 性能要求

| 指标 | 目标 |
|------|------|
| 启动时间 | < 2 秒 |
| 大表（>10万行）首屏加载 | < 3 秒（分页） |
| 并发查询 | 同时加载 ≥3 个表无卡顿 |
| 内存占用 | 空闲状态 < 100MB |

| 阶段 | 目标 | 重点 |

| **第一阶段** | 基础功能可用 | 多线程查询、浏览、编辑功能跑通，UI 从简 |

## 8. 测试策略

- **单元测试**：覆盖核心模块（连接管理、查询执行、配置管理）
- **集成测试**：验证完整登录 → 浏览 → 编辑流程
- **CLI 后端**：提供非 GUI 命令行接口，方便自动化测试和功能验证
- **手动测试**：各阶段完成后进行真实 MySQL 环境测试
