# SQL-Viewer Lite — 项目规格说明书

## 1. 项目概述

**SQL-Viewer Lite** 是一款轻量、高速的多数据库查看与编辑工具，基于 Python 桌面技术栈开发。旨在为开发者和数据管理员提供一个简洁、高效的数据库浏览体验，支持多数据库、多连接、多进程并发查询、虚拟滚动、数据编辑和现代化 UI 设计。

### 1.1 项目目标

- 提供类 Navicat / DBeaver 的多数据库浏览与编辑体验
- 支持 MySQL、PostgreSQL、SQLite、SQL Server 等主流数据库
- 使用 QSS 实现现代化、前端风格的用户界面
- 保持轻量级，启动快、占用资源少
- 同时提供 GUI 和 CLI/后端接口，方便测试和集成

### 1.2 技术栈

| 层级 | 技术选型 |
|------|---------|
| 编程语言 | Python 3.8+ |
| GUI 框架 | PyQt5 |
| 数据库驱动 | PyMySQL / psycopg2 / sqlite3 / pymssql |
| 样式方案 | QSS (Qt Style Sheets) |
| 并发模型 | multiprocessing.Process (推荐) / QThread (备选) |
| 虚拟滚动 | QTableView + QAbstractTableModel |
| 构建/打包 | PyInstaller (第三阶段) |

## 2. 核心功能

### 2.1 用户登录

- 用户通过输入数据库连接信息进行身份验证
- 支持多种数据库类型：MySQL、PostgreSQL、SQLite、SQL Server
- 登录成功后获取服务器上的数据库列表
- 支持保存/管理多个连接配置（本地加密存储）
- 连接失败时提供清晰的错误提示

### 2.2 多连接管理

- 同时保持多个数据库连接
- 在不同连接之间快速切换
- 每个连接独立维护数据库/表树
- 连接状态实时监控
- 跨库查询支持

### 2.3 数据库浏览

- 登录后展示当前数据库服务器上所有可用数据库
- 用户选择一个数据库后，加载该库下所有数据表
- 支持数据库/表的搜索和筛选
- 展示表的基本信息（表名、行数、引擎、字符集等）
- CLI 文件树风格显示层级结构（├── └── 树线 + Emoji 图标）

### 2.4 数据查看

- 以表格形式展示选中表的数据内容
- 支持虚拟滚动，大数据量无卡顿
- 支持分页加载，避免大数据表一次性加载导致卡顿
- 支持列排序（升序/降序）
- 支持按列筛选/搜索（可横向滚动）
- 支持查看表结构（字段名、类型、约束、索引等）

### 2.5 数据编辑

- 支持直接在表格中编辑单元格数据
- 支持新增行、删除行操作
- 支持批量编辑后统一提交
- 编辑前自动备份原始数据，支持撤销
- 所有写操作通过事务处理，确保数据一致性

### 2.6 多进程查询

- 使用 `multiprocessing.Process` 实现真正的并行查询
- 避免 Python GIL 限制，多核 CPU 利用率更高
- 使用 `multiprocessing.Queue` 进程间通信
- 主界面保持响应，不因后台查询而阻塞
- 支持查询任务的取消操作

### 2.7 虚拟滚动

- 基于 `QTableView` + `QAbstractTableModel` 实现
- 只渲染可见区域，内存高效
- 支持百万行数据流畅显示
- 保持现有编辑、排序、筛选功能

### 2.8 键盘快捷键

- 支持 16 个常用快捷键
- 主要快捷键：Ctrl+C/V/S, F5, Ctrl+F, Ctrl+N
- 快捷键配置可持久化
- 鼠标中键关闭标签页

### 2.9 SQL 执行器

- 提供简易 SQL 输入框，支持手动执行自定义 SQL
- 展示查询结果和执行信息
- 支持查询历史记录
- 支持选择目标连接执行查询

## 3. 支持的数据库

### 3.1 数据库矩阵

| 数据库 | 驱动 | 优先级 | 状态 | 特殊说明 |
|--------|------|--------|------|---------|
| MySQL | PyMySQL | P0 | ✅ 已支持 | 完整功能支持 |
| PostgreSQL | psycopg2 | P1 | ⬜ 待开发 | 支持 schema、ILIKE 等 |
| SQLite | sqlite3 (内置) | P1 | ⬜ 待开发 | 无需主机/端口/用户名 |
| SQL Server | pymssql | P2 | ⬜ 待开发 | 支持 TOP、IDENTITY 等 |
| Oracle | oracledb | P3 | ⬜ 待开发 | 优先级较低 |

### 3.2 功能降级策略

- 某些数据库不支持的功能优雅降级
- 显示功能支持状态提示
- SQL 方言自动适配（LIMIT/OFFSET 语法差异）

## 4. 架构设计

### 4.1 模块划分

```
sql_viewer_lite/
├── main.py                 # 程序入口（含 HiDPI 配置）
├── ui/                     # UI 相关模块
│   ├── __init__.py
│   ├── login_window.py     # 登录窗口（支持多数据库类型）
│   ├── table_view.py       # 数据表格视图（含筛选控件）
│   ├── table_structure.py  # 表结构查看组件
│   ├── sql_editor.py       # SQL 执行器（支持多连接）
│   ├── data_table_model.py # 虚拟滚动数据模型
│   ├── virtual_table_view.py # 虚拟滚动视图
│   └── styles/             # QSS 样式文件
│       ├── main.qss        # 深色主题
│       └── light.qss       # 浅色主题
├── core/                   # 核心业务逻辑
│   ├── __init__.py
│   ├── db_connection.py    # 统一连接管理
│   ├── db_worker.py        # 多进程查询工作者
│   ├── connection_pool.py  # 连接池
│   ├── ssh_tunnel.py       # SSH 隧道
│   └── drivers/            # 数据库驱动适配器
│       ├── __init__.py
│       ├── base.py         # 抽象基类 DatabaseDriver
│       ├── mysql.py        # MySQL 驱动
│       ├── postgresql.py   # PostgreSQL 驱动
│       ├── sqlite.py       # SQLite 驱动
│       └── mssql.py        # SQL Server 驱动
├── models/                 # 数据模型
│   ├── __init__.py
│   └── connection.py       # 连接配置模型（支持多数据库类型）
├── utils/                  # 工具函数
│   ├── __init__.py
│   ├── config_manager.py   # 连接配置的持久化管理
│   ├── encryption.py       # 密码加密存储
│   ├── theme_manager.py    # 主题管理
│   ├── i18n.py             # 国际化
│   └── shortcuts.py        # 键盘快捷键
├── backend/                # 非 GUI 后端
│   ├── __init__.py
│   └── cli.py              # 命令行接口
├── tests/                  # 单元测试
│   ├── __init__.py
│   ├── test_db_connection.py
│   ├── test_config_manager.py
│   ├── test_encryption.py
│   └── test_data_table_model.py
└── requirements.txt
```

### 4.2 数据流

```
用户操作 → UI 层 → 信号发射 → ProcessWorker → 数据库驱动 → 数据库查询 → 结果返回 → UI 更新
```

### 4.3 关键设计原则

- **MVC 分层**：UI 层、业务逻辑层、数据访问层分离
- **驱动抽象**：统一的 DatabaseDriver 接口，支持多数据库
- **信号驱动**：跨进程通信使用 multiprocessing.Queue + QTimer 轮询
- **连接池**：数据库连接池统一管理，避免连接泄漏
- **延迟加载**：大表数据分页加载，元数据按需获取
- **虚拟滚动**：只渲染可见区域，内存高效

## 5. UI 设计规范

### 5.1 整体风格

- 采用 QSS 实现现代化扁平设计，参考 VS Code 风格
- 深色主题（默认）和浅色主题
- 圆角卡片式布局，适当的阴影和间距
- 响应式布局，支持窗口缩放

### 5.2 界面布局

```
┌─────────────────────────────────────────────────────┐
│  工具栏：连接选择 | 新增行 | 删除行 | 提交 | 撤销    │
├────────────────┬────────────────────────────────────┤
│                │  筛选控件（可横向滚动）              │
│   侧边栏       ├────────────────────────────────────┤
│   - 搜索框     │         数据表格区域                │
│   - 数据库树   │   (多 Tab 页，每个表一个 Tab)       │
│     - 表列表   │                                    │
│                ├────────────────────────────────────┤
│                │  分页控件                           │
├────────────────┴────────────────────────────────────┤
│  状态栏：连接信息 | 当前表 | 行数 | 查询耗时          │
└─────────────────────────────────────────────────────┘
```

### 5.3 交互规范

- 登录窗口居中显示，支持数据库类型选择
- 左侧数据库/表树形结构支持搜索过滤
- CLI 文件树风格显示层级（├── └── 树线 + Emoji）
- 鼠标中键点击标签页可关闭
- 数据表格支持键盘快捷键（Ctrl+C/V/S 等）
- 加载数据时显示加载指示器
- HiDPI 显示器支持（高分辨率不模糊）

---

## 6. 安全性

- 密码在本地存储时使用 AES 加密
- 连接信息不明文写入配置文件
- SQL 执行器限制危险操作（如 DROP、TRUNCATE），需二次确认
- 连接超时自动断开，避免长连接资源占用

---

## 7. 性能要求

| 指标 | 目标 |
|------|------|
| 启动时间 | < 2 秒 |
| 大表（>10万行）首屏加载 | < 3 秒（虚拟滚动） |
| 并发查询 | 同时加载 ≥3 个表无卡顿 |
| 内存占用 | 空闲状态 < 100MB |
| 100万行数据 | 加载 < 5 秒，内存 < 200MB |

## 8. 开发阶段

| 阶段 | 目标 | 重点 |
|------|------|------|
| **第一阶段** | 基础功能可用 | 多线程查询、浏览、编辑功能跑通，UI 从简 |

| **第三阶段** | 打包发布 | PyInstaller 打包为独立可执行程序 |
| **附加优化** | 高级特性 | 虚拟滚动、快捷键、多进程、HiDPI |
| **多连接管理** | 多连接 | 多连接并行、连接切换、跨库查询 |
| **多数据库支持** | 多数据库 | PostgreSQL、SQLite、SQL Server |

## 9. 测试策略

- **单元测试**：覆盖核心模块（连接管理、查询执行、配置管理、数据模型）
- **集成测试**：验证完整登录 → 浏览 → 编辑流程
- **CLI 后端**：提供非 GUI 命令行接口，方便自动化测试和功能验证
- **手动测试**：各阶段完成后进行真实数据库环境测试
